# Support

Thanks for using the ECZ-ID MCP Verifier. Please route your question to the right place.

> Do not include secrets, credentials, private customer data, prompts, tool payloads or
> private logs in any public issue or discussion.

## Where to go

| Topic | Where |
|---|---|
| MCP host / server configuration | [Open an issue](https://github.com/Ecocitizenz/ecz-id-mcp-verifier/issues/new/choose) (MCP host setup) — or run `npx @ecocitizenz/ecz-id-mcp-verifier@0.8.1 --print-mcp-config` |
| CLI usage | [Issues](https://github.com/Ecocitizenz/ecz-id-mcp-verifier/issues) — see `--help` and the README |
| GitHub Action / CI | [Issues](https://github.com/Ecocitizenz/ecz-id-mcp-verifier/issues) — see the Action usage in the README |
| Node library integration | [Integration question](https://github.com/Ecocitizenz/ecz-id-mcp-verifier/issues/new/choose) or the [Developer Gateway](https://developers.ecocitizenz.com) |
| Improve a target's public Resolver posture (operators) | [TrustOps](https://trustops.ecocitizenz.com/start) |
| Verify public Resolver proof | [Resolver](https://resolver.ecocitizenz.org) |
| Security concern / vulnerability | See [SECURITY.md](SECURITY.md) — report privately, never in a public issue |

## Quick self-help

```sh
npx @ecocitizenz/ecz-id-mcp-verifier@0.8.1 --doctor        # confirm a healthy install
npx @ecocitizenz/ecz-id-mcp-verifier@0.8.1 --capabilities  # what it does and does not do
```

The Resolver is the public proof surface; npm, GitHub and Registry listings are discovery,
not proof. Re-check before reliance. Local policy decides.
